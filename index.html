The Problem

Normal websites were not built for AI

Search engines built decades of infrastructure for crawling and ranking HTML. AI assistants, LLM search, and AI browsers are building something new — and they need structured, machine-readable context that most websites simply don't provide.

AI assistants cite structured sources

LLMs prefer structured, unambiguous data over dense HTML. A business with a clean agent-context.json is easier to cite accurately.

AI search doesn't index like Google

AI search systems look for intent, identity, and trustworthiness signals that go beyond keywords and backlinks.

AI browsers are arriving

Agentic browsers and AI-driven discovery tools will increasingly navigate the web on behalf of users. Businesses that are machine-readable will have a structural advantage.

The AFO Package

10 files. One AI-readable business layer.

AFO generates and installs a set of structured files tailored to your business. Each file serves a specific purpose in AI discoverability.

  • ✓ llms.txt — plain-text LLM identity summary /llms.txt
  • ✓ agent-context.json — structured business identity /.well-known/
  • ✓ agent-actions.json — available actions for AI agents /.well-known/
  • ✓ agent-policy.json — compliance and claim boundaries /.well-known/
  • ✓ context-cookie.json — concise identity memory object /.well-known/
  • ✓ sitemap-agent.xml — agent-optimized sitemap /sitemap-agent.xml
  • ✓ robots.txt — crawler permissions with AI annotations /robots.txt
  • ✓ Dashboard JSON — before/after audit data /data/
  • ✓ Proof report — benchmarkable visibility record /data/reports/
  • ✓ Schema-structured page markup — JSON-LD business identity index.html
